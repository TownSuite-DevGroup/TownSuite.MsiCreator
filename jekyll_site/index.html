Step 01 · Prerequisites

Install the WiX toolset

MsiCreator builds on top of WiX 4. Install it as a global .NET tool and make sure your PATH includes the dotnet tools directory before running a build.

PowerShell 
dotnet tool install --global wix --version 4.0.6
setx PATH "%PATH%;C:\Users\<USERNAME>\.dotnet\tools"
$dotnetToolsPath = "$env:USERPROFILE\.dotnet\tools"
$env:PATH += ";$dotnetToolsPath"
Step 02 · Quick start

Build your first installer

Point MsiCreator at the folder containing your compiled binaries. A -SrcZip archive works as a source too.

.\TownSuite.MsiCreator.exe `
  -CompanyName "<CompanyName>" `
  -Product "<Product>" `
  -Version "<Version>" `
  -SrcBinDirectory "<SrcBinDirectory>" `
  -OutputDirectory "<OutputDirectory>" `
  -MainExecutable "<MainExecutable>" `
  -ProductGuid "<ProductGuid>" `
  -Platform "x64"
.\TownSuite.MsiCreator.exe `

Alternative output

NSIS .exe installer

Same arguments as an MSI build — just add -OutputType "exe" to emit a self-contained NSIS executable.

PowerShell 
.\TownSuite.MsiCreator.exe `
  -OutputType "exe" `
  -CompanyName "<CompanyName>" -Product "<Product>" `
  -Version "<Version>" `
  -SrcBinDirectory "<SrcBinDirectory>" `
  -OutputDirectory "<OutputDirectory>" `
  -MainExecutable "<MainExecutable>" `
  -ProductGuid "<ProductGuid>" -Platform "x64"
Install behavior

Per-user or per-machine

The generated MSI installs per user by default — double-click it or run it from PowerShell. Pass ALLUSERS=1 to install machine-wide.

Per user (default)

msiexec /i "MyProduct.msi"
Install path
C:\Users\[User]\AppData\Local\Programs\[CompanyName]\[Product]

Per machine (admin)

msiexec /i "MyProduct.msi" ALLUSERS=1
Install path
C:\Program Files\[CompanyName]\[Product]
Pipelines

Pin a release in CI/CD

Use the bundled download_release.ps1 script to fetch a specific version of MsiCreator during your build, then invoke it against your artifacts.

build-pipeline.ps1 
# 1 — fetch a pinned msicreator release into your workspace
.\download_release.ps1

# 2 — package your freshly built artifacts
.\TownSuite.MsiCreator.exe `
  -CompanyName "Example" -Product "Example" `
  -Version "1.0.0" `
  -SrcBinDirectory ".\artifacts" `
  -OutputDirectory ".\dist" `
  -MainExecutable "Example.exe" `
  -ProductGuid "ee85df4f-6f19-4d3a-85bd-91b94657650f" `
  -Platform "x64"

A -SrcZip archive can be used in place of -SrcBinDirectory.
